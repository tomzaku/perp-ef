Explain the JavaScript event loop, including:

1. The call stack and how synchronous code executes
2. The task queue (macrotask queue) — setTimeout, setInterval, I/O
3. The microtask queue — Promises, queueMicrotask, MutationObserver
4. Execution order: all microtasks drain before next macrotask
5. requestAnimationFrame and rendering in the browser
6. Common interview output-order questions

## Examples

**Input:** console.log('1'); setTimeout(() => console.log('2'), 0); Promise.resolve().then(() => console.log('3')); console.log('4');
**Output:** 1, 4, 3, 2
*Sync code (1, 4) runs first, then microtasks (3), then macrotasks (2).*


## Solution

```js
// ============================================
// 1. Event Loop Overview
// ============================================
/*
  The JavaScript runtime has:

  +----------------------------+
  |       Call Stack            |  <- Executes synchronous code
  +-------------+--------------+
                |
  +-------------v--------------+
  |    Microtask Queue          |  <- Promise.then, queueMicrotask
  |    (drains completely)      |     MutationObserver
  +-------------+--------------+
                |
  +-------------v--------------+
  |    Macrotask Queue          |  <- setTimeout, setInterval, I/O
  |    (one task per cycle)     |     setImmediate (Node)
  +----------------------------+

  Event Loop Cycle:
  1. Execute all synchronous code on the call stack
  2. Drain ALL microtasks (including newly added ones)
  3. Render (browser only: rAF, layout, paint)
  4. Pick ONE macrotask and execute it
  5. Go to step 2
*/

// ============================================
// 2. Basic Execution Order
// ============================================
console.log('1 - sync');

setTimeout(() => {
  console.log('2 - macrotask');
}, 0);

Promise.resolve().then(() => {
  console.log('3 - microtask');
});

console.log('4 - sync');

// Output:
// 1 - sync
// 4 - sync
// 3 - microtask
// 2 - macrotask

// ============================================
// 3. Microtasks Drain Completely
// ============================================
console.log('Start');

setTimeout(() => console.log('setTimeout'), 0);

Promise.resolve()
  .then(() => {
    console.log('Promise 1');
    // This microtask is added during microtask processing
    // It runs BEFORE the setTimeout macrotask!
    Promise.resolve().then(() => {
      console.log('Promise 2 (nested)');
    });
  });

console.log('End');

// Output:
// Start
// End
// Promise 1
// Promise 2 (nested)   <- nested microtask runs before setTimeout
// setTimeout

// ============================================
// 4. Complex Execution Order
// ============================================
console.log('script start');

  });
}, 0);

setTimeout(() => {

}, 0);

Promise.resolve()
  .then(() => {
    console.log('promise 1');

// promise 2
// setTimeout 1
// promise inside setTimeout  <- microtask drains before next macrotask

  console.log('async after await');

// 10. Practical: Avoiding Long Task Blocking

// Break up heavy work to keep UI responsive

function processLargeArray(array, processFn, chunkSize = 100) {
  let index = 0;

  function processChunk() {
    const end = Math.min(index + chunkSize, array.length);

    for (; index < end; index++) {
      processFn(array[index], index);
    }

    if (index < array.length) {
      // Yield to the event loop, let browser render
      setTimeout(processChunk, 0);
    }
  }

  processChunk();
}
```

## Explanation

CONCEPT: Event Loop, Call Stack, and Task Queues

```
EVENT LOOP ARCHITECTURE:

┌──────────────────────────────┐
│         CALL STACK           │  ← Executes synchronous code
│  (one frame at a time)       │
└─────────────┬────────────────┘
              │ empty?
              ▼
┌──────────────────────────────┐
│      MICROTASK QUEUE         │  ← Promise callbacks, queueMicrotask
│  (drain ALL before moving on)│     process.nextTick (Node)
└─────────────┬────────────────┘
              │ empty?
              ▼
┌──────────────────────────────┐
│      MACROTASK QUEUE         │  ← setTimeout, setInterval, I/O
│  (process ONE, then check    │     requestAnimationFrame
│   microtasks again)          │
└──────────────────────────────┘
```

EXECUTION ORDER PUZZLE:

```
console.log('1');
setTimeout(() => console.log('2'), 0);
Promise.resolve().then(() => console.log('3'));
console.log('4');

Step-by-step:
Stack                  Microtask Q    Macrotask Q    Output
─────                  ───────────    ───────────    ──────
log('1')               []             []             1
setTimeout(cb, 0)      []             [cb→'2']
Promise.then(cb)       [cb→'3']       [cb→'2']
log('4')               [cb→'3']       [cb→'2']       1, 4
── stack empty, drain microtasks ──
log('3')               []             [cb→'2']       1, 4, 3
── microtasks empty, take one macrotask ──
log('2')               []             []             1, 4, 3, 2
```

KEY RULE: Microtasks ALWAYS run before the next macrotask.
Promise.then → microtask. setTimeout → macrotask. This is why 3 prints before 2.
