Compare three approaches for real-time data:

**Polling:** Client repeatedly asks server for updates at intervals.
**Server-Sent Events (SSE):** Server pushes events to client over HTTP. One-way (server → client).
**WebSocket:** Full-duplex communication over a persistent connection. Two-way.

When to use each? How to handle reconnection and scaling?

## Solution

```js
// ════════════════════════════════════════════
// 1. POLLING
// ════════════════════════════════════════════

class Poller {
  #intervalId = null;
  #url;
  #intervalMs;
  #onData;

  constructor(url, intervalMs, onData) {
    this.#url = url;
    this.#intervalMs = intervalMs;
    this.#onData = onData;
  }

  start() {
    this.#poll(); // Immediate first poll
    this.#intervalId = setInterval(() => this.#poll(), this.#intervalMs);
  }

  async #poll() {
    try {
      const res = await fetch(this.#url);
      const data = await res.json();
      this.#onData(data);
    } catch (err) {
      console.error('Poll failed:', err);
    }
  }

  stop() {
    clearInterval(this.#intervalId);
  }
}

// Usage: poll every 5 seconds
const poller = new Poller('/api/notifications', 5000, (data) => {
  console.log('New notifications:', data);
});
poller.start();
// poller.stop();

// ════════════════════════════════════════════
// 2. SERVER-SENT EVENTS (SSE)
// ════════════════════════════════════════════

class SSEClient {
  #eventSource = null;
  #url;
  #onMessage;
  #reconnectMs = 1000;

  constructor(url, onMessage) {
    this.#url = url;
    this.#onMessage = onMessage;
  }

  connect() {
    this.#eventSource = new EventSource(this.#url);

    this.#eventSource.onmessage = (event) => {
      const data = JSON.parse(event.data);
      this.#onMessage(data);
    };

    // Named events
    this.#eventSource.addEventListener('notification', (event) => {
      this.#onMessage({ type: 'notification', data: JSON.parse(event.data) });
    });

    this.#eventSource.onerror = () => {
      console.log('SSE disconnected. Reconnecting in', this.#reconnectMs, 'ms');
      this.#eventSource.close();
      setTimeout(() => this.connect(), this.#reconnectMs);
      this.#reconnectMs = Math.min(this.#reconnectMs * 2, 30000);
    };

    this.#eventSource.onopen = () => {
      this.#reconnectMs = 1000; // Reset backoff on successful connect
    };
  }

  close() {
    this.#eventSource?.close();
  }
}

const sse = new SSEClient('/api/events', (data) => {
  console.log('SSE event:', data);
});
sse.connect();

// ════════════════════════════════════════════
// 3. WEBSOCKET
// ════════════════════════════════════════════

class WebSocketClient {
  #ws = null;
  #url;
  #handlers = new Map();
  #reconnectMs = 1000;
  #maxReconnect = 30000;
  #messageQueue = [];

  constructor(url) {
    this.#url = url;
  }

  connect() {
    this.#ws = new WebSocket(this.#url);

    this.#ws.onopen = () => {
      console.log('WebSocket connected');
      this.#reconnectMs = 1000;
      // Flush queued messages
      while (this.#messageQueue.length > 0) {
        this.#ws.send(this.#messageQueue.shift());
      }
    };

    this.#ws.onmessage = (event) => {
      const { type, data } = JSON.parse(event.data);
      const handlers = this.#handlers.get(type) || [];
      handlers.forEach(fn => fn(data));
    };

    this.#ws.onclose = () => {
      console.log('Reconnecting in', this.#reconnectMs, 'ms...');
      setTimeout(() => this.connect(), this.#reconnectMs);
      this.#reconnectMs = Math.min(this.#reconnectMs * 2, this.#maxReconnect);
    };

    this.#ws.onerror = (err) => {
      console.error('WebSocket error:', err);
    };
  }

  on(type, handler) {
    if (!this.#handlers.has(type)) this.#handlers.set(type, []);
    this.#handlers.get(type).push(handler);
    return () => {
      const arr = this.#handlers.get(type);
      arr.splice(arr.indexOf(handler), 1);
    };
  }

  send(type, data) {
    const message = JSON.stringify({ type, data });
    if (this.#ws?.readyState === WebSocket.OPEN) {
      this.#ws.send(message);
    } else {
      this.#messageQueue.push(message); // Queue if not connected
    }
  }

  close() {
    this.#ws?.close();
  }
}

const ws = new WebSocketClient('wss://api.example.com/ws');
ws.connect();
ws.on('chat', (msg) => console.log('Chat:', msg));
ws.on('typing', (data) => console.log(data.user, 'is typing...'));
ws.send('chat', { text: 'Hello!' });
```

## ELI5

Imagine you're waiting for a friend to tell you something happened. You have three ways to find out:

**Polling = calling every 5 minutes:** "Any news?" "No." "Any news?" "No." "Any news?" "Yes! She had the baby!" Simple, but wasteful — you spent 50 calls for 1 piece of news.

**SSE = asking them to call you when something happens:** You say "Text me when there's news." They text you. You don't have to do anything. One-way communication (they tell you things; you don't text back).

**WebSocket = leaving a phone call open:** Both of you can talk anytime. You can say things, they can say things, all in real time. Feels like a two-way walkie-talkie.

```
                    Polling         SSE             WebSocket
Direction:          Client→Server   Server→Client   Both ways
Protocol:           HTTP            HTTP            WS (special)
Connection:         New each time   One persistent  One persistent
Latency:            High (interval) Low (instant)   Lowest
Scaling:            Easy (stateless) Moderate        Hard (stateful)

Best for:
  Polling     → checking email every 5 min, low frequency updates
  SSE         → Twitter feed, stock ticker, notifications, live scores
  WebSocket   → chat, multiplayer game, Google Docs live editing
```

**Reconnection** is essential for SSE and WebSocket. Network drops happen. The right strategy is exponential backoff:

```
Connection drops:
  Wait 1s  → reconnect → still fails
  Wait 2s  → reconnect → still fails
  Wait 4s  → reconnect → succeeds! ✓
  (Don't hammer the server with instant retries)

SSE auto-reconnects built-in (browser handles it)
WebSocket needs manual reconnection logic
```
