# Pilot Test Guide

## Purpose

The pilot test validates the questionnaire before full-scale data collection. It ensures:
1. Questions are clear and unambiguous
2. Response options are comprehensive
3. Survey flow and skip logic work correctly
4. Completion time is reasonable (target: 10-15 minutes)
5. Scales produce acceptable reliability (Cronbach's α ≥ 0.70)

---

## Pilot Sample

- **Size:** 15-20 students (not included in the final sample)
- **Criteria:** Same population as the main study (undergraduate students at the same university)
- **Diversity:** Include students from different years, disciplines, and English levels

---

## Pilot Test Procedure

### Phase 1: Think-Aloud (5 students)

Ask 5 students to complete the survey while talking through their thought process:

- Do they understand every question?
- Are they confused by any wording?
- Do the response options cover their situation?
- Are any questions sensitive or uncomfortable?

**Record notes on:**
| Question | Issue Reported | Suggested Fix |
|----------|---------------|---------------|
| | | |

### Phase 2: Full Pilot (15-20 students)

1. Distribute the online questionnaire
2. Record completion time for each respondent
3. After completion, ask the following debrief questions:

**Debrief Questions:**
1. Were any questions unclear or confusing? Which ones?
2. Were there any questions where none of the answer choices applied to you?
3. Was the survey too long, too short, or about right?
4. Were there any topics related to GenAI in academic writing that the survey didn't ask about?
5. Did the survey flow feel logical?
6. Any other comments or suggestions?

---

## Pilot Data Analysis

### 1. Completion Metrics
- Average completion time: _____ minutes
- Completion rate: _____ %
- Drop-off points (where respondents stopped): _____

### 2. Reliability Analysis
Run Cronbach's alpha for each scale:

| Scale | # Items | Cronbach's α | Action |
|-------|---------|-------------|--------|
| Language Improvement (C1) | 6 | | |
| Writing Process (C2) | 6 | | |
| Learning & Confidence (C3) | 5 | | |
| Plagiarism Understanding (D1) | 6 | | |
| Ethical Concerns (D2) | 6 | | |
| Behavioral Intentions (D3) | 4 | | |

**If α < 0.70:**
- Check "alpha if item deleted" column
- Consider rewording or removing problematic items
- Re-run reliability after changes

### 3. Item Analysis
For each Likert item, check:
- Mean and SD (avoid ceiling/floor effects — mean shouldn't be > 4.5 or < 1.5)
- Item-total correlation (should be ≥ 0.30)
- Missing data rate per item

### 4. Response Distribution
- Check for items where > 80% chose the same response (low variance — may not be useful)
- Check "Other" responses in multiple-select items — add popular write-in options

---

## Revision Checklist

After the pilot, review and revise:

- [ ] Reword any confusing questions
- [ ] Add missing response options identified by participants
- [ ] Remove or merge redundant items
- [ ] Fix any skip logic issues
- [ ] Adjust survey length if needed
- [ ] Update translations (if bilingual version)
- [ ] Re-verify reliability targets are met

---

## Bilingual Considerations

Since respondents are Vietnamese students:

### Option A: English-only questionnaire
- Simpler, no translation needed
- Risk: Students with lower English proficiency may misunderstand questions
- Mitigated by: Pilot testing for comprehension

### Option B: Bilingual (English + Vietnamese)
- More inclusive and accessible
- Requires: Professional translation + back-translation
- Recommended if: Target sample includes students below B2 level

### Translation Process (if bilingual)
1. Forward translation: English → Vietnamese (by a bilingual researcher)
2. Back-translation: Vietnamese → English (by a different bilingual person)
3. Compare original and back-translated versions
4. Resolve discrepancies
5. Pilot test both versions
