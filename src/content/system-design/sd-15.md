**Optimistic updates** make the UI feel instant by updating immediately before the server confirms. If the server rejects the change, the UI rolls back.

**Use cases:**
- Like/unlike buttons (toggle instantly)
- Adding items to a list
- Editing text/names
- Drag-and-drop reordering

**Challenges:**
- Rolling back on server error
- Handling concurrent modifications
- Managing temporary IDs for new items
- Keeping UI consistent during rollback

Implement optimistic update patterns.

## Solution

```js
// ════════════════════════════════════════════
// OPTIMISTIC UPDATE PATTERN
// ════════════════════════════════════════════

class OptimisticStore {
  #state;
  #pendingUpdates = new Map(); // id → { previous, current }
  #listeners = new Set();

  constructor(initialState) {
    this.#state = initialState;
  }

  getState() { return this.#state; }

  subscribe(fn) {
    this.#listeners.add(fn);
    return () => this.#listeners.delete(fn);
  }

  #notify() {
    this.#listeners.forEach(fn => fn(this.#state));
  }

  // Optimistic update with automatic rollback
  async optimisticUpdate(id, updateFn, serverFn) {
    const previous = JSON.parse(JSON.stringify(this.#state));

    // 1. Apply optimistically
    this.#state = updateFn(this.#state);
    this.#pendingUpdates.set(id, { previous });
    this.#notify();

    try {
      // 2. Send to server
      const serverResult = await serverFn();

      // 3. Apply server response (may differ slightly from optimistic)
      if (serverResult) {
        this.#state = serverResult;
        this.#notify();
      }

      this.#pendingUpdates.delete(id);
      return { success: true };
    } catch (err) {
      // 4. Rollback on failure
      const pending = this.#pendingUpdates.get(id);
      if (pending) {
        this.#state = pending.previous;
        this.#pendingUpdates.delete(id);
        this.#notify();
      }
      return { success: false, error: err.message };
    }
  }

  hasPendingUpdates() {
    return this.#pendingUpdates.size > 0;
  }
}

// ════════════════════════════════════════════
// EXAMPLE: Todo list with optimistic CRUD
// ════════════════════════════════════════════

const todoStore = new OptimisticStore([
  { id: 1, text: 'Buy groceries', done: false },
  { id: 2, text: 'Walk the dog', done: true },
]);

// Toggle todo completion (optimistic)
async function toggleTodo(todoId) {
  return todoStore.optimisticUpdate(
    'toggle_' + todoId,
    // Optimistic update
    (state) => state.map(todo =>
      todo.id === todoId ? { ...todo, done: !todo.done } : todo
    ),
    // Server call
    async () => {
      const res = await fetch('/api/todos/' + todoId + '/toggle', { method: 'PATCH' });
      if (!res.ok) throw new Error('Failed to toggle');
      return res.json(); // Updated list from server
    }
  );
}

// Add todo (optimistic with temp ID)
async function addTodo(text) {
  const tempId = 'temp_' + Date.now();

  return todoStore.optimisticUpdate(
    'add_' + tempId,
    // Optimistic: add with temp ID
    (state) => [...state, { id: tempId, text, done: false }],
    // Server: replace temp ID with real ID
    async () => {
      const res = await fetch('/api/todos', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ text }),
      });
      if (!res.ok) throw new Error('Failed to add');
      const newTodo = await res.json();
      const state = todoStore.getState();
      return state.map(t => t.id === tempId ? newTodo : t);
    }
  );
}

// Delete todo (optimistic)
async function deleteTodo(todoId) {
  return todoStore.optimisticUpdate(
    'delete_' + todoId,
    (state) => state.filter(t => t.id !== todoId),
    async () => {
      const res = await fetch('/api/todos/' + todoId, { method: 'DELETE' });
      if (!res.ok) throw new Error('Failed to delete');
      return null; // Optimistic state is correct
    }
  );
}

// Subscribe to updates
todoStore.subscribe((todos) => {
  console.log('Todos:', todos.map(t => (t.done ? '✓' : '○') + ' ' + t.text));
});

// Usage — UI updates INSTANTLY
toggleTodo(1); // ✓ Buy groceries (instant, server confirms later)
addTodo('New task'); // Shows immediately with temp ID
deleteTodo(2); // Disappears instantly
```

## ELI5

Imagine you're liking a tweet. You tap the heart button. Without optimistic updates, nothing happens for 200ms while the request travels to Twitter's server and back — then the heart turns red.

**With optimistic updates**, the heart turns red **instantly** when you tap it. The server request happens in the background. If it fails, the heart unturns red.

```
Without optimistic updates:
  Tap heart → wait 200ms → ❤️ red
  (Feels slow and unresponsive)

With optimistic updates:
  Tap heart → ❤️ red INSTANTLY (optimistic)
              → server request in background
              → server confirms → ✓ done
              → if server fails → 🤍 undo (rollback)
```

**The challenge is rollback.** You need to remember the "before" state in case things go wrong:

```
Toggle todo "Buy milk" from incomplete → complete:

  1. Save old state: { id: 1, done: false, text: "Buy milk" }
  2. Apply optimistically: show as done ✓ (immediately)
  3. Send PATCH /todos/1 to server
  4a. Server succeeds → great, optimistic was right ✓
  4b. Server fails → restore old state: show as incomplete again
      + show error: "Failed to update. Try again."
```

**Temp IDs** for new items solve the "what ID does this have?" problem:

```
Add a new todo:
  1. Create with temp ID: { id: "temp_1234", text: "New task" }
     → shows in list immediately with temp ID
  2. POST /todos → server returns { id: 99, text: "New task" }
  3. Replace temp_1234 with 99 in the list
  → seamless! User never notices the swap.

  If POST fails: remove temp_1234 from list entirely (rollback)
```
