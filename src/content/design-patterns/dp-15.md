The Facade Pattern provides a simplified interface to a complex subsystem. It doesn't add new functionality вҖ” it makes existing functionality easier to use by hiding complexity behind a clean API.

**Frontend use cases:**
- Wrapping complex browser APIs (Web Audio, Canvas, WebGL)
- Unifying multiple microservice calls behind a single function
- Simplifying a complex library's API for common use cases
- jQuery is essentially a facade over the DOM API

**When to use:**
- When a subsystem has many interdependent classes
- When you want a simple default interface for common tasks
- When you want to decouple client code from subsystem details

## Solution

```js
// ============================
// Facade over multiple API services
// ============================
class UserApi {
  async getProfile(id) { return { id, name: 'Alice', email: 'alice@test.com' }; }
}

class OrderApi {
  async getOrders(userId) { return [{ id: 1, total: 99.99 }]; }
}

class NotificationApi {
  async getUnread(userId) { return [{ id: 1, message: 'Welcome!' }]; }
}

class PreferencesApi {
  async getPrefs(userId) { return { theme: 'dark', lang: 'en' }; }
}

// Facade: one call instead of four
class UserDashboardFacade {
  constructor() {
    this.userApi = new UserApi();
    this.orderApi = new OrderApi();
    this.notifApi = new NotificationApi();
    this.prefsApi = new PreferencesApi();
  }

  async getDashboardData(userId) {
    const [profile, orders, notifications, preferences] = await Promise.all([
      this.userApi.getProfile(userId),
      this.orderApi.getOrders(userId),
      this.notifApi.getUnread(userId),
      this.prefsApi.getPrefs(userId),
    ]);

    return {
      profile,
      orders,
      notifications,
      preferences,
      summary: {
        totalOrders: orders.length,
        unreadCount: notifications.length,
        displayName: profile.name,
      }
    };
  }
}

const dashboard = new UserDashboardFacade();
dashboard.getDashboardData(1).then(data => {
  console.log(data.summary);
  // { totalOrders: 1, unreadCount: 1, displayName: 'Alice' }
});

// ============================
// DOM Manipulation Facade (mini jQuery)
// ============================
const $ = {
  select(selector) {
    return document.querySelector(selector);
  },
  selectAll(selector) {
    return [...document.querySelectorAll(selector)];
  },
  on(selector, event, handler) {
    const el = typeof selector === 'string' ? this.select(selector) : selector;
    if (el) el.addEventListener(event, handler);
    return () => el?.removeEventListener(event, handler);
  },
  css(selector, styles) {
    const el = typeof selector === 'string' ? this.select(selector) : selector;
    if (el) Object.assign(el.style, styles);
  },
  addClass(selector, ...classes) {
    const el = typeof selector === 'string' ? this.select(selector) : selector;
    if (el) el.classList.add(...classes);
  },
  hide(selector) { this.css(selector, { display: 'none' }); },
  show(selector) { this.css(selector, { display: '' }); },
};

// Clean facade over verbose DOM API
// $.on('#btn', 'click', () => $.hide('#modal'));
// $.css('#header', { background: 'blue', color: 'white' });

// ============================
// Complex Media API Facade
// ============================
class MediaFacade {
  #stream = null;
  #recorder = null;
  #chunks = [];

  async startRecording() {
    this.#stream = await navigator.mediaDevices.getUserMedia({ audio: true });
    this.#recorder = new MediaRecorder(this.#stream);
    this.#chunks = [];
    this.#recorder.ondataavailable = (e) => this.#chunks.push(e.data);
    this.#recorder.start();
  }

  async stopRecording() {
    return new Promise(resolve => {
      this.#recorder.onstop = () => {
        const blob = new Blob(this.#chunks, { type: 'audio/webm' });
        this.#stream.getTracks().forEach(t => t.stop());
        resolve({ blob, url: URL.createObjectURL(blob) });
      };
      this.#recorder.stop();
    });
  }

  play(url) { return new Audio(url).play(); }
}
```

## Explanation

**The core problem Facade solves:**

Without a facade, every caller must know about *all* the internal pieces of a subsystem and orchestrate them correctly. That creates high coupling вҖ” a change inside the subsystem ripples out to every caller.

```
Without Facade вҖ” every component does its own orchestration:

  DashboardPage:
    const profile = await userApi.getProfile(id);      // must know UserApi
    const orders  = await orderApi.getOrders(id);       // must know OrderApi
    const notifs  = await notifApi.getUnread(id);       // must know NotifApi
    const prefs   = await prefsApi.getPrefs(id);        // must know PrefsApi
    // must know Promise.all trick
    // must know how to assemble summary

  ProfilePage: same 4 imports, same orchestration...
  SettingsPage: same again...

  вҶ’ If OrderApi changes its method signature, you update it everywhere.
```

```
With Facade вҖ” one import, one call:

  DashboardPage:
    const data = await dashboard.getDashboardData(id);
    // done. No knowledge of subsystem internals.

  ProfilePage: same one call.
  SettingsPage: same one call.

  вҶ’ OrderApi changes? Update only the Facade.
```

**Facade vs Adapter вҖ” the key distinction:**

| | Adapter | Facade |
|---|---|---|
| Wraps | One class | An entire subsystem |
| Purpose | Convert interface A вҶ’ B | Simplify complexity |
| Adds new behaviour? | No | Yes (orchestration logic) |
| Hides the original? | No, adapts it | No, subsystem still accessible |

**Facade does not prevent direct access.** Advanced users can still call `userApi`, `orderApi` etc. directly. The Facade is a convenience layer, not a lock.

**Layered Facades** are common in large apps:

```
UI Components
     вҶ“
Service Facade (getDashboardData)   вҶҗ what we built
     вҶ“
API Layer (UserApi, OrderApi...)
     вҶ“
HTTP / fetch
```

Each layer is a facade over the one below it. React Query's `useQuery` is a facade over fetch + caching + loading states + error handling.

## ELI5

Imagine you want to watch a movie at home. Without a Facade, you'd have to:
1. Turn on the TV
2. Switch TV input to HDMI-2
3. Turn on the receiver
4. Set receiver to "Movie" mode
5. Dim the smart lights
6. Close the smart blinds
7. Press play on the streaming box

That's 7 steps across 4 different devices. You have to know how everything works together.

**The Facade** is a single "Movie Mode" button on a universal remote. Press it вҖ” everything above happens automatically. You don't need to know the steps. You don't even need to know those devices exist.

The devices are still there if you want to control them individually (direct subsystem access). But for the common case, one button is all you need.
