The Composite pattern lets you **treat individual objects and groups of objects uniformly** through a shared interface. It's the pattern behind every tree structure: file systems, DOM trees, organisation charts, React component trees.

**The key insight:** A `Folder` and a `File` both implement `getSize()`. A folder's `getSize()` sums its children recursively. The caller just calls `getSize()` — it doesn't care whether it has a leaf or a subtree.

**Frontend use cases:**
- React component tree — a `<Form>` is a composite of `<Input>`, `<Button>`, `<Section>` components
- DOM tree — every node (element or text) can be traversed with the same interface
- File system UI — folders and files both have `name`, `size`, `render()`
- Menu systems — menu items and submenus share the same `render()` interface

## Solution

```js
// Component interface — shared by both leaf and composite
class FileSystemItem {
  constructor(name) { this.name = name; }
  getSize()   { throw new Error('getSize() not implemented'); }
  render(indent = 0) { throw new Error('render() not implemented'); }
}

// Leaf — no children
class File extends FileSystemItem {
  constructor(name, size) {
    super(name);
    this.size = size;
  }
  getSize()          { return this.size; }
  render(indent = 0) { console.log(`${'  '.repeat(indent)}📄 ${this.name} (${this.size}kb)`); }
}

// Composite — has children, delegates to them
class Folder extends FileSystemItem {
  constructor(name) {
    super(name);
    this.children = [];
  }

  add(item)    { this.children.push(item); return this; }
  remove(item) { this.children = this.children.filter(c => c !== item); }

  // Recursive — works for any depth
  getSize() {
    return this.children.reduce((sum, child) => sum + child.getSize(), 0);
  }

  render(indent = 0) {
    console.log(`${'  '.repeat(indent)}📁 ${this.name} (${this.getSize()}kb)`);
    this.children.forEach(child => child.render(indent + 1));
  }
}

// Build the tree
const root = new Folder('project');
const src = new Folder('src');
const assets = new Folder('assets');

src.add(new File('index.ts', 12));
src.add(new File('app.tsx', 34));
assets.add(new File('logo.png', 120));
assets.add(new File('icon.svg', 8));

root.add(src).add(assets).add(new File('package.json', 2));

root.render();
// 📁 project (176kb)
//   📁 src (46kb)
//     📄 index.ts (12kb)
//     📄 app.tsx (34kb)
//   📁 assets (128kb)
//     📄 logo.png (120kb)
//     📄 icon.svg (8kb)
//   📄 package.json (2kb)

root.getSize(); // 176 — same call on root or any subtree
```

## Explanation

```
                   root (Folder)
                  /      |       \
            src(F)    assets(F)  package.json(File)
            /  \        /   \
      index.ts app.tsx logo.png icon.svg
      (File)   (File)  (File)   (File)
```

Every node — whether a `File` (leaf) or `Folder` (composite) — responds to `getSize()` and `render()`. The `Folder` delegates to its children, which may themselves be folders. The recursion terminates at `File` leaves.

**In React**, every component is a composite. `<App>` renders `<Header>` and `<Main>`. `<Main>` renders `<Sidebar>` and `<Content>`. React's reconciler calls `render()` on each — it doesn't distinguish leaves from composites.

## Diagram

```
          Component (interface)
          ├── getSize()
          └── render()
               /          \
          File            Folder
        (Leaf)          (Composite)
        ──────          ──────────
        getSize()       children: []
        → this.size     getSize() → sum children
                        render()  → recurse children
                        add() / remove()
```

## ELI5

A company has employees and departments. You ask "what's the total salary cost?" — you can ask an individual employee (leaf), or you can ask a whole department (composite) and it adds up all its employees and sub-departments for you. Same question, either way. You don't need to know whether you're talking to a person or a whole team.
