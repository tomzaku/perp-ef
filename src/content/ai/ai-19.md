**What are the key AI SDKs and frameworks, and how do you choose between them?**

The AI development ecosystem includes provider SDKs, orchestration frameworks, and UI libraries. Understanding the tradeoffs helps you pick the right tool for your application.

## Solution

```text
AI SDK LANDSCAPE
════════════════

Layer 1: PROVIDER SDKs (Low-level, direct API access)
──────────────────────────────────────────────────────
Anthropic SDK    → Claude models (messages API, tool use, streaming)
OpenAI SDK       → GPT models (chat completions, assistants, functions)
Google GenAI SDK → Gemini models (multimodal, grounding)

Layer 2: ORCHESTRATION FRAMEWORKS (High-level abstractions)
──────────────────────────────────────────────────────────
LangChain        → Chains, agents, memory, retrieval
LlamaIndex       → Data ingestion, indexing, RAG pipelines
CrewAI           → Multi-agent orchestration
Haystack         → NLP pipelines, RAG, document processing

Layer 3: UI & APPLICATION SDKs
──────────────────────────────
Vercel AI SDK    → Streaming UI, React hooks, multi-provider
Claude Code SDK  → Agent SDK for building CLI/coding agents


PROVIDER SDK PATTERNS
═════════════════════

// Anthropic SDK — Clean, minimal API
import Anthropic from "@anthropic-ai/sdk";
const client = new Anthropic();

const response = await client.messages.create({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  system: "You are a helpful coding assistant.",
  messages: [{ role: "user", content: "Explain closures in JS" }],
  // Tool use
  tools: [{
    name: "search_docs",
    description: "Search documentation",
    input_schema: { type: "object", properties: { query: { type: "string" } } }
  }]
});

// Streaming
const stream = client.messages.stream({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  messages: [{ role: "user", content: "Write a React component" }]
});

for await (const event of stream) {
  if (event.type === "content_block_delta") {
    process.stdout.write(event.delta.text);
  }
}


// OpenAI SDK — Similar pattern, different API shape
import OpenAI from "openai";
const openai = new OpenAI();

const completion = await openai.chat.completions.create({
  model: "gpt-4o",
  messages: [{ role: "user", content: "Explain closures" }],
  tools: [{
    type: "function",
    function: {
      name: "search_docs",
      parameters: { type: "object", properties: { query: { type: "string" } } }
    }
  }]
});


VERCEL AI SDK — FULL-STACK AI

// Multi-provider with unified API
import { generateText, streamText } from "ai";
import { anthropic } from "@ai-sdk/anthropic";
import { openai } from "@ai-sdk/openai";

// Same API, swap providers
const result = await generateText({
  model: anthropic("claude-sonnet-4-6"),  // or openai("gpt-4o")
  prompt: "Explain closures",
  tools: {
    searchDocs: {
      description: "Search documentation",
      parameters: z.object({ query: z.string() }),
      execute: async ({ query }) => searchDocs(query)
    }
  }
});

// React hooks for streaming UI
// app/chat/page.tsx
"use client";
import { useChat } from "@ai-sdk/react";

export default function Chat() {
  const { messages, input, handleInputChange, handleSubmit } = useChat();

  return (
    <div>
      {messages.map(m => (
        <div key={m.id}>
          {m.role}: {m.content}
        </div>
      ))}
      <form onSubmit={handleSubmit}>
        <input value={input} onChange={handleInputChange} />
      </form>
    </div>
  );
}

────────────────┼─────────────┼────────────┼────────────┤
Complexity      │ Low         │ Medium     │ High       │

CHOOSING THE RIGHT TOOL
═══════════════════════

Use DIRECT SDK when:
  ✓ You want full control over API calls
  ✓ Simple use case (chat, completion, tool use)
  ✓ You want minimal dependencies
  ✓ Building a library or framework yourself

Use VERCEL AI SDK when:
  ✓ Building a Next.js / React web app
  ✓ Need streaming UI out of the box
  ✓ Want to switch providers easily
  ✓ Full-stack TypeScript project

Use LANGCHAIN when:
  ✓ Complex RAG pipelines with many data sources
  ✓ Need chains of LLM calls with memory
  ✓ Rapid prototyping of agent architectures
  ✓ Large team familiar with the framework

Use AGENT SDK when:
  ✓ Building autonomous coding/task agents
  ✓ Need tool orchestration with agentic loops
  ✓ Building CLI tools or development assistants


ANTI-PATTERNS
═════════════

✗ Using LangChain for a simple chat app (over-engineering)
✗ Building your own streaming from scratch when Vercel AI SDK exists
✗ Coupling to one provider's SDK when you might switch later
✗ Ignoring error handling — LLM APIs fail, have rate limits, timeout
✗ Not implementing retries with exponential backoff
```

## Explanation

The AI SDK landscape is evolving rapidly, but the core decision comes down to **how much abstraction you need**.

**The simplicity principle:** Start with the lowest level of abstraction that meets your needs. Direct SDKs give you the most control with the least complexity. Only reach for frameworks when you genuinely need their features.

**Key trends:**

1. **Provider convergence** — APIs are becoming more similar (messages, tools, streaming). Multi-provider SDKs like Vercel AI SDK capitalize on this
2. **Simpler > Complex** — The industry is moving away from heavy frameworks (early LangChain) toward lighter, composable abstractions
3. **Full-stack integration** — AI SDKs increasingly include UI primitives (streaming hooks, generative UI components)
4. **Agent-first** — New SDKs are designed around agentic patterns from the start, not just request-response

**Practical advice:** For most web applications, the Vercel AI SDK with a provider SDK gives you the best balance of power and simplicity. For CLI tools or agents, a direct SDK with a thin orchestration layer is usually sufficient.

## ELI5

Think of AI SDKs like cooking tools. A direct SDK is like having basic pots and pans — you have full control and can make anything, but you do all the work. A framework like LangChain is like a bread maker — it does a lot for you, but you're limited to what it supports. The Vercel AI SDK is like a well-equipped kitchen with nice appliances — it helps with common tasks but still lets you cook however you want. Pick the one that matches what you're cooking!
