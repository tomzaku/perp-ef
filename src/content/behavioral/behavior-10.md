Frontend evolves fast. The interviewer wants to know you can keep up without burning out, and that you apply what you learn — not just collect knowledge.

## Examples

**Input:** "How do you stay up to date with new technologies and trends?"
**Output:** A concrete system showing intentional, applied learning.

## Solution

```text
Framework: System → Application → Sharing

"I have a three-layer approach that keeps me current without
overwhelming myself:

LAYER 1: CURATED INTAKE (30 min/day)
I follow a focused list — not everything. My sources are:
- 3-4 newsletters (JavaScript Weekly, React Status, Node Weekly)
- A few key people on Twitter/Bluesky who filter signal from noise
- The 'This Week in React' digest

I timebox this to my morning coffee. If something catches my
eye, I bookmark it for deeper reading later.

LAYER 2: HANDS-ON EXPERIMENTATION (weekends, ~2hrs)
Reading about a technology isn't the same as using it. When
something seems promising, I build a small project. For example,
when React Server Components were announced, I built a tiny
blog app to understand the mental model before we evaluated
it for work.

This is how I discovered that our team could benefit from
Suspense boundaries for our data-heavy dashboards — I'd already
experimented with the pattern.

LAYER 3: TEACHING AND SHARING
I run a bi-weekly 'Tech Radar' session on my team where someone
presents a technology they've explored. Preparing a presentation
forces you to truly understand something. I've given talks on
Signals (reactivity), Edge Runtime trade-offs, and container
queries.

APPLICATION:
Last quarter, my experimentation with View Transitions API
directly led to our team adopting it for page transitions.
It improved our Interaction to Next Paint score by 40% and
the UX felt dramatically smoother. That wouldn't have happened
if I'd only read about it."
```

## Explanation

**What makes a strong answer:**
1. **You have a system** — not random browsing
2. **You apply what you learn** — connect it to real work impact
3. **You share knowledge** — multiplier effect on the team
4. **You're selective** — you filter, not firehose
5. **You can point to results** — learning led to concrete improvements

**What interviewers want to see:**
- You're **self-directed** — no one has to push you to learn
- You're **practical** — you don't chase every shiny thing
- You **share with the team** — learning isn't just for yourself
- You have **taste** — you can evaluate what's worth adopting vs. what's hype
