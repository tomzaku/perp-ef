**Stale-While-Revalidate (SWR)** is a caching strategy where:
1. Return cached data immediately (stale)
2. Fetch fresh data in the background (revalidate)
3. Update the cache and UI when fresh data arrives

This provides instant UI feedback while keeping data fresh. Libraries like SWR (Vercel) and React Query (TanStack) implement this.

Implement a SWR-like cache from scratch with:
- Automatic revalidation
- Cache expiry
- Deduplication of concurrent requests
- Optimistic updates

## Solution

```js
// ════════════════════════════════════════════
// SWR Cache Implementation
// ════════════════════════════════════════════

class SWRCache {
  #cache = new Map();
  #inflight = new Map(); // Deduplicate concurrent requests
  #subscribers = new Map();
  #defaultTTL = 60000; // 1 minute

  // Subscribe to cache key updates
  subscribe(key, callback) {
    if (!this.#subscribers.has(key)) this.#subscribers.set(key, new Set());
    this.#subscribers.get(key).add(callback);
    return () => this.#subscribers.get(key)?.delete(callback);
  }

  #notify(key, data) {
    this.#subscribers.get(key)?.forEach(cb => cb(data));
  }

  // Core SWR fetch
  async fetch(key, fetcher, options = {}) {
    const { ttl = this.#defaultTTL, revalidate = true } = options;
    const cached = this.#cache.get(key);

    // Return stale data immediately
    if (cached) {
      const isStale = Date.now() - cached.timestamp > ttl;
      if (!isStale) return cached.data;

      // Stale: return cached but revalidate in background
      if (revalidate) {
        this.#revalidate(key, fetcher);
      }
      return cached.data;
    }

    // No cache: fetch and wait
    return this.#fetchAndCache(key, fetcher);
  }

  async #fetchAndCache(key, fetcher) {
    // Deduplicate: if same key is already being fetched, reuse the promise
    if (this.#inflight.has(key)) {
      return this.#inflight.get(key);
    }

    const promise = fetcher()
      .then(data => {
        this.#cache.set(key, { data, timestamp: Date.now() });
        this.#notify(key, { data, isValidating: false });
        return data;
      })
      .finally(() => {
        this.#inflight.delete(key);
      });

    this.#inflight.set(key, promise);
    return promise;
  }

  async #revalidate(key, fetcher) {
    this.#notify(key, { data: this.#cache.get(key)?.data, isValidating: true });
    return this.#fetchAndCache(key, fetcher);
  }

  // Optimistic update: update cache immediately, revert on error
  async mutate(key, data, fetcher) {
    const previous = this.#cache.get(key)?.data;

    // Optimistically update
    this.#cache.set(key, { data, timestamp: Date.now() });
    this.#notify(key, { data, isValidating: true });

    try {
      const result = await fetcher();
      this.#cache.set(key, { data: result, timestamp: Date.now() });
      this.#notify(key, { data: result, isValidating: false });
      return result;
    } catch (err) {
      // Rollback on failure
      if (previous !== undefined) {
        this.#cache.set(key, { data: previous, timestamp: Date.now() });
        this.#notify(key, { data: previous, isValidating: false, error: err });
      }
      throw err;
    }
  }

  // Invalidate cache entry
  invalidate(key) {
    this.#cache.delete(key);
  }

  // Clear all cache
  clear() {
    this.#cache.clear();
  }
}

// ════════════════════════════════════════════
// Usage
// ════════════════════════════════════════════

const cache = new SWRCache();

// Fetch with SWR pattern
async function getUser(id) {
  return cache.fetch(
    'user:' + id,
    () => fetch('/api/users/' + id).then(r => r.json()),
    { ttl: 30000 } // 30s TTL
  );
}

// Subscribe to updates
cache.subscribe('user:1', ({ data, isValidating }) => {
  console.log('User data:', data, 'Refreshing:', isValidating);
});

// First call: fetches from network
const user = await getUser(1);

// Second call within 30s: returns cached instantly
const cachedUser = await getUser(1);

// Optimistic update
await cache.mutate(
  'user:1',
  { ...user, name: 'New Name' }, // Optimistic data
  () => fetch('/api/users/1', {
    method: 'PUT',
    body: JSON.stringify({ name: 'New Name' }),
  }).then(r => r.json())
);
```

## ELI5

Imagine a news app. When you open it, you want to see news **right now** — not a loading spinner. But you also want the **latest** news, not yesterday's.

**SWR (Stale-While-Revalidate)** does both:

```
First visit:
  Open app → loading spinner → fetch news → show news → save to cache
  (Normal: first time always needs a fetch)

Second visit (30 seconds later):
  Open app → instantly show CACHED news (stale, but fast!)
              → simultaneously fetch fresh news in background
              → when fresh news arrives → update the screen
  
  Result: zero waiting time + always fresh data
```

It's like getting yesterday's newspaper from your doorstep the moment you open the door, while the morning delivery person is already on their way with today's edition.

**Deduplication** prevents multiple components from fetching the same thing:

```
Without deduplication:
  Component A loads → fetches /api/user
  Component B loads → fetches /api/user  ← same request!
  Component C loads → fetches /api/user  ← same request again!
  = 3 network calls for the same data

With SWR deduplication:
  Component A loads → fetches /api/user → in-flight promise saved
  Component B loads → "already fetching!" → reuses the same promise
  Component C loads → "already fetching!" → reuses the same promise
  = 1 network call, all 3 components get the result ✓
```

**Optimistic mutation** = show the change before the server confirms it, rollback if it fails. Combined with SWR, it makes your app feel instant.
