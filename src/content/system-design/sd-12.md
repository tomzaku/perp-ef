Understanding browser caching is essential for web performance. HTTP cache headers control how browsers and CDNs cache responses.

**Key headers:**
- `Cache-Control`: max-age, no-cache, no-store, immutable
- `ETag`: content hash for conditional requests
- `Last-Modified`: timestamp for conditional requests

**Strategies:**
- **Immutable assets:** Cache forever (files with content hash in name)
- **Stale-while-revalidate:** Serve stale, fetch fresh in background
- **No cache:** Always validate with server (ETag/Last-Modified)

Explain caching strategies and implement a Service Worker cache.

## Solution

```js
// ════════════════════════════════════════════
// HTTP CACHE HEADERS (server-side configuration)
// ════════════════════════════════════════════

const cacheStrategies = {
  // Static assets with hash in filename (main.a1b2c3.js)
  // Cache forever — filename changes when content changes
  immutableAssets: {
    'Cache-Control': 'public, max-age=31536000, immutable',
    // 1 year, never revalidate
  },

  // HTML pages — always check for updates
  htmlPages: {
    'Cache-Control': 'no-cache',
    // Browser caches but ALWAYS revalidates with ETag/Last-Modified
    // NOT the same as no-store (which doesn't cache at all)
  },

  // API responses — cache briefly, revalidate in background
  apiResponses: {
    'Cache-Control': 'public, max-age=60, stale-while-revalidate=300',
    // Serve cached for 60s, then serve stale while fetching fresh for up to 5min
  },

  // Sensitive data — never cache
  sensitiveData: {
    'Cache-Control': 'no-store, private',
    // Not cached anywhere — not browser, not CDN
  },

  // CDN-cacheable, private to user
  userSpecific: {
    'Cache-Control': 'private, max-age=300',
    // Cached by browser (5min) but NOT by CDN
  },
};

// ════════════════════════════════════════════
// CONDITIONAL REQUESTS (304 Not Modified)
// ════════════════════════════════════════════

// Server sends:
// ETag: "abc123"
// Last-Modified: Tue, 01 Jan 2024 00:00:00 GMT

// Browser sends (on revalidation):
// If-None-Match: "abc123"
// If-Modified-Since: Tue, 01 Jan 2024 00:00:00 GMT

// Server responds: 304 Not Modified (no body = saves bandwidth)

// ════════════════════════════════════════════
// SERVICE WORKER: Cache-First Strategy
// ════════════════════════════════════════════

// service-worker.js
const CACHE_NAME = 'app-v1';
const STATIC_ASSETS = ['/index.html', '/app.js', '/styles.css'];

// Install: pre-cache static assets
// self.addEventListener('install', (event) => {
//   event.waitUntil(
//     caches.open(CACHE_NAME).then(cache => cache.addAll(STATIC_ASSETS))
//   );
// });

// Fetch: cache-first for assets, network-first for API
// self.addEventListener('fetch', (event) => {
//   const url = new URL(event.request.url);
//
//   if (url.pathname.startsWith('/api/')) {
//     // Network-first for API calls
//     event.respondWith(networkFirst(event.request));
//   } else {
//     // Cache-first for static assets
//     event.respondWith(cacheFirst(event.request));
//   }
// });

async function cacheFirst(request) {
  const cached = await caches.match(request);
  if (cached) return cached;
  const response = await fetch(request);
  const cache = await caches.open(CACHE_NAME);
  cache.put(request, response.clone());
  return response;
}

async function networkFirst(request) {
  try {
    const response = await fetch(request);
    const cache = await caches.open(CACHE_NAME);
    cache.put(request, response.clone());
    return response;
  } catch {
    return caches.match(request); // Fallback to cache (offline)
  }
}

async function staleWhileRevalidate(request) {
  const cached = await caches.match(request);
  const fetchPromise = fetch(request).then(response => {
    caches.open(CACHE_NAME).then(cache => cache.put(request, response.clone()));
    return response;
  });
  return cached || fetchPromise;
}

console.log('Cache strategies loaded');
```

## ELI5

Imagine you have a water bottle on your desk. Getting water from the bottle is instant. Walking to the kitchen takes 30 seconds.

**Browser caching** = keeping files in your water bottle (browser cache) instead of fetching from the server every time.

```
Without caching:
  Load page → fetch HTML, CSS, JS, images → renders
  Refresh page → fetch HTML, CSS, JS, images AGAIN → renders
  (Even though nothing changed!)

With caching:
  Load page → fetch everything → browser saves copies
  Refresh page → browser uses saved copies → instant!
  (Only re-fetches if the file has actually changed)
```

**Different files need different freshness strategies:**

```
🟢 Cache forever (static assets with hash in filename):
   app.a1b2c3.js — the filename changes when the code changes
   → Cache for 1 year: browser never needs to check

🟡 Always check for updates (HTML):
   index.html — contains links to the latest JS/CSS files
   → no-cache: browser checks if content changed (gets 304 if same)
   → quick check, fast if unchanged

🔴 Never cache (sensitive user data):
   /api/account/balance
   → no-store: browser never saves this at all

🔵 Serve stale, update in background (API responses):
   /api/posts — usually fine if 60 seconds old
   → stale-while-revalidate: show old data instantly, fetch fresh in background
```

**Service Workers** let you take full control — even work offline. They sit between your app and the network, intercepting every request.

```
Cache-first:  Check cache → serve from cache → (also fetch fresh for next time)
Network-first: Try network → if offline → serve from cache
```
