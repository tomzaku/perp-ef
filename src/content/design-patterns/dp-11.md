The Builder Pattern separates the construction of a complex object from its representation, allowing the same construction process to create different representations. It's especially useful when an object has many optional parameters.

**Frontend use cases:**
- Query/URL builders
- HTTP request builders
- Configuration objects with many options
- Test data builders (creating test fixtures)

**Key advantage:** Method chaining provides a fluent, readable API. Instead of a constructor with 10 parameters, the builder guides you step by step.

## Solution

```js
// ============================
// Query Builder
// ============================
class QueryBuilder {
  #table = '';
  #conditions = [];
  #orderFields = [];
  #limitVal = null;
  #offsetVal = null;
  #selectedFields = ['*'];

  from(table) { this.#table = table; return this; }
  select(...fields) { this.#selectedFields = fields; return this; }

  where(field, operator, value) {
    this.#conditions.push({ field, operator, value });
    return this;
  }

  orderBy(field, direction = 'ASC') {
    this.#orderFields.push(`${field} ${direction}`);
    return this;
  }

  limit(n) { this.#limitVal = n; return this; }
  offset(n) { this.#offsetVal = n; return this; }

  build() {
    let query = `SELECT ${this.#selectedFields.join(', ')} FROM ${this.#table}`;
    if (this.#conditions.length) {
      const clauses = this.#conditions.map(c => `${c.field} ${c.operator} '${c.value}'`).join(' AND ');
      query += ` WHERE ${clauses}`;
    }
    if (this.#orderFields.length) query += ` ORDER BY ${this.#orderFields.join(', ')}`;
    if (this.#limitVal !== null) query += ` LIMIT ${this.#limitVal}`;
    if (this.#offsetVal !== null) query += ` OFFSET ${this.#offsetVal}`;
    return query;
  }
}

const query = new QueryBuilder()
  .from('users')
  .select('id', 'name', 'email')
  .where('age', '>=', 18)
  .where('active', '=', true)
  .orderBy('name', 'ASC')
  .limit(10)
  .offset(20)
  .build();

console.log(query);
// SELECT id, name, email FROM users WHERE age >= '18' AND active = 'true' ORDER BY name ASC LIMIT 10 OFFSET 20

// ============================
// HTTP Request Builder
// ============================
class RequestBuilder {
  #url = '';
  #method = 'GET';
  #headers = {};
  #body = null;
  #params = {};
  #timeout = 5000;

  setUrl(url) { this.#url = url; return this; }
  setMethod(m) { this.#method = m; return this; }
  addHeader(k, v) { this.#headers[k] = v; return this; }
  setBody(data) { this.#body = data; return this; }
  addParam(k, v) { this.#params[k] = v; return this; }
  setTimeout(ms) { this.#timeout = ms; return this; }

  build() {
    const qs = new URLSearchParams(this.#params).toString();
    const fullUrl = qs ? `${this.#url}?${qs}` : this.#url;
    const options = {
      method: this.#method,
      headers: this.#headers,
      signal: AbortSignal.timeout(this.#timeout),
    };
    if (this.#body) {
      options.body = JSON.stringify(this.#body);
      options.headers['Content-Type'] = 'application/json';
    }
    return { url: fullUrl, options };
  }

  async execute() {
    const { url, options } = this.build();
    return fetch(url, options).then(r => r.json());
  }
}

const request = new RequestBuilder()
  .setUrl('https://api.example.com/users')
  .setMethod('POST')
  .addHeader('Authorization', 'Bearer token123')
  .setBody({ name: 'Alice' })
  .addParam('notify', 'true')
  .setTimeout(10000)
  .build();

console.log(request.url); // https://api.example.com/users?notify=true

// ============================
// Test Data Builder
// ============================
class UserBuilder {
  #user = { id: 1, name: 'Default User', email: 'default@test.com', role: 'viewer', active: true };

  withId(id) { this.#user.id = id; return this; }
  withName(name) { this.#user.name = name; return this; }
  withEmail(email) { this.#user.email = email; return this; }
  asAdmin() { this.#user.role = 'admin'; return this; }
  asInactive() { this.#user.active = false; return this; }
  build() { return { ...this.#user }; }
}

const admin = new UserBuilder().withName('Admin').asAdmin().build();
const inactive = new UserBuilder().withName('Old User').asInactive().build();
console.log(admin);    // { id: 1, name: 'Admin', role: 'admin', active: true }
console.log(inactive); // { id: 1, name: 'Old User', role: 'viewer', active: false }
```

## Explanation

**The problem the Builder solves:** Complex objects with many optional parameters become unmanageable with a single constructor. `new Request(url, method, headers, body, timeout, retries, auth, cache, ...)` — by position 7, nobody knows which argument is which.

**The Builder provides a fluent interface:** You chain method calls, each setting one parameter, and call `.build()` at the end. Each step is named and readable. Optional steps can be skipped without passing `null` or `undefined`.

**Method chaining works by returning `this`:** Each setter method returns the builder instance, enabling `builder.setUrl().setMethod().build()`.

**Immutability advantage:** The `.build()` call typically returns a new immutable object. The builder is mutable during construction; the result is not.

**Three patterns in the code:**
1. **Query builder** — constructs SQL/API queries step by step
2. **Request builder** — builds fetch configurations cleanly
3. **Test data builder** — creates test fixtures with sensible defaults, override only what matters

**Builder vs Factory:**
- Factory: creates one of several types based on input → "which type?"
- Builder: constructs one complex object step by step → "how to configure?"

## Diagram

```
BUILDER PATTERN — step-by-step construction:

  Instead of:
    new Request('/api', 'POST', headers, body, 30000, 3, true, false)
    ← which arg is which? impossible to read!

  Builder way:
    ┌─────────────────────────────────────────┐
    │           RequestBuilder                │
    │                                         │
    │  .setUrl('/api/users')    → this        │
    │  .setMethod('POST')       → this        │
    │  .setBody({ name: 'Bob'}) → this        │
    │  .setTimeout(30000)       → this        │
    │  .withAuth(token)         → this        │
    │  .build()                 → Request {}  │
    └─────────────────────────────────────────┘

  Each step:
    1. Sets one clearly named property
    2. Returns `this` to enable chaining
    3. Is optional (sensible defaults exist)

  .build() validates and returns the final, immutable object.


FLUENT CHAIN VISUALIZATION:

  new RequestBuilder()
       │
       ▼ .setUrl()        → builder (url set)
       ▼ .setMethod()     → builder (method set)
       ▼ .setBody()       → builder (body set)
       ▼ .withRetry(3)    → builder (retries set)
       ▼ .build()         → { url, method, body, retries, ... }
                             FINAL IMMUTABLE OBJECT
```

## ELI5

Imagine ordering a custom sandwich at Subway. You don't hand them a pre-written list of 20 ingredients at once. You build it step by step:

"I want sourdough bread... turkey... lettuce... no onions... extra pickles... toasted... with mustard."

Each choice is one step. You skip the things you don't want. At the end you get your finished sandwich.

```
Without Builder (constructor hell):
  new Sandwich(
    'sourdough',  // bread
    'turkey',     // protein
    true,         // lettuce
    false,        // tomato
    false,        // onion
    true,         // pickles
    true,         // toasted
    'mustard',    // sauce
    false         // mayo
  );
  ← which true/false is which? impossible to read!

With Builder (step by step):
  new SandwichBuilder()
    .bread('sourdough')
    .protein('turkey')
    .addLettuce()
    .extraPickles()
    .toasted()
    .sauce('mustard')
    .build();  ← returns the sandwich

  Each step is readable. Skip steps you don't want.
  Add new options without changing existing callers.
```

The Builder is especially useful for **test data** — build objects with sensible defaults, override only what the test cares about:
```js
// Test only cares about the role, not every user field
const adminUser = new UserBuilder().withRole('admin').build();
```
